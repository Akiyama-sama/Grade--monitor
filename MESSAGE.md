# `-` 项目展示字段提取与展示优先级分析

本文档基于当前目录下真实存在的 `-` 项目代码提取而来，主要依据：

- `-/src/App.tsx`
- `-/src/index.css`
- `-/metadata.json`

目标：
- 提取该项目**界面中实际展示的所有字段**
- 按照代码中的布局，说明这些字段的**展示优先级**、**视觉占比**、**信息层级**
- 作为后续页面优化的唯一字段依据

---

## 1. 项目总体信息

- 项目名：**智能废钢判级监控大屏**
- 项目描述：**简约科技现代风的废钢判级智能监控大屏，集成基础信息、报警监测、视频画面及判级结果分析。**
- 页面结构：
  - 顶部 TopBar
  - 中间主内容区（12列栅格）
    - 左栏：报警监测 + 实时信息流
    - 中栏：综合结果 + 双视频 + 历史抓拍库
    - 右栏：基础信息 + 判级深度分析
  - 底部 Footer
  - 若干弹层：详情弹层、图片轮播弹层、确认弹层

---

# 2. 页面布局与视觉优先级

## 2.1 页面栅格占比

主内容区使用 12 列布局：

- **左栏：2/12 ≈ 16.7%**
- **中栏：7/12 ≈ 58.3%**
- **右栏：3/12 = 25%**

这意味着代码中的视觉优先级非常明确：

### 一级核心区（最高优先级）
1. **中栏（58.3%）**
   - 综合结果（Hero）
   - 视频画面（主监控内容）
   - 历史抓拍库

### 二级分析区（高优先级）
2. **右栏（25%）**
   - 基础信息
   - 判级深度分析
   - 厚度占比
   - 料型占比

### 三级辅助区（中优先级）
3. **左栏（16.7%）**
   - 报警监测
   - 实时信息流

### 四级系统信息区（低优先级）
4. **顶部 / 底部**
   - 系统标题
   - 当前时间
   - 系统状态
   - 引擎、延迟、授权状态

---

## 2.2 垂直层级优先级

按代码结构，页面从上到下的展示优先级如下：

1. **TopBar**：系统身份 + 当前时间
2. **中栏 Hero 综合结果**：当前判级状态 / 智能级别分布 / 厚度 / 扣杂总量 / 操作按钮
3. **视频画面**：枪机 / 球机
4. **右栏基础信息 + 判级深度分析**
5. **左栏报警监测 + 实时信息流**
6. **历史抓拍库**
7. **Footer 系统运行信息**
8. **Modal 弹层内容**（触发时优先级提升到最高）
