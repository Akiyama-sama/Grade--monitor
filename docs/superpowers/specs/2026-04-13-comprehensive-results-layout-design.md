# 综合结果区排版调整设计说明

日期：2026-04-13  
项目：废钢判级监控大屏原型  
适用代码目录：`/Users/mac/Develop/work/yongyou/废钢判级监控大屏原型`

## 1. 背景

当前中间上方的综合结果区承载了以下信息：

- 判级状态
- 三个操作按钮
- 智能级别判断
- 智能扣杂雷达
- 扣杂总量与明细摘要

这些内容目前同时竞争同一行高，导致视觉重心分散，雷达图右侧说明信息过重，按钮与状态信息之间也缺少明确分层。

用户要求本次只重排该区域，不调整其他区域结构，并特别要求：

- 雷达图不要配太多文字
- 雷达图仅保留图本体与各角子项名称
- 判级状态、智能综合厚度、智能级别分布应放在左侧上方一块
- 三个按钮位于左侧下方，但与左侧上方共用同一个外层盒子高度
- 雷达区与左侧综合结果区在同一个外层盒子里横向排布
- 尽量不要使用额外卡片包裹，而是用分割线来划分区域
- 排版在当前容器内稳定，不出现溢出

## 2. 目标与非目标

### 2.1 目标

- 重新组织综合结果区的视觉层次
- 提高中上区域信息的扫读效率
- 让按钮区从信息摘要区中独立出来
- 降低雷达图的说明密度
- 保持整体风格、色彩语言、交互行为与周边区域一致

### 2.2 非目标

- 不改左侧栏和右侧分析栏
- 不改视频区和抓拍图区结构
- 不新增业务功能
- 不更改按钮行为
- 不修改整体三栏布局比例

## 3. 信息结构重组

综合结果区保持为一个整体盒子，并在盒子内部做左右分区。

### 3.1 外层结构

外层仍是一个综合结果大盒子，但内部改为左右两区：

- 左区为“综合结果 + 按钮区”
- 右区为“雷达区”

左右两区在同一外层容器中横向排布，并保持整体高度一致。

### 3.2 左区结构

左区再分为上下两层，并用横向分割线隔开：

- 上层：判级状态 + 综合厚度 + 智能级别
- 下层：三个操作按钮

#### 左区上层

左区上层承载 3 个核心摘要模块：

1. 判级状态
2. 智能综合厚度
3. 智能级别分布

其中：

- `判级状态` 继续作为最大字号主标题
- 车牌与当前监控点 badge 保留，但从属到状态信息中
- `智能综合厚度` 改为单独的大数字摘要，例如 `6`
- `智能综合厚度` 不显示单位，不展示厚度区间
- `智能级别分布` 仅负责展示候选级别和概率，例如：
  - `重1 20%`
  - `重2 70%`
- 级别分布继续保留条形概率条，但与综合厚度一起位于状态右侧信息区中

#### 左区下层

三个按钮位于左区下层：

- `结束判级`
- `异常处理`
- `中途离开`

要求：

- 三个按钮在一行内平均分布
- 按钮与上层信息通过分割线隔开
- 按钮区高度被纳入左区总高度中，而不是另起一块独立高度

### 3.3 右区雷达结构

右区仅保留紧凑雷达图展示，不再做卡片式包裹。

右区仅保留：

- 雷达图本体
- 图上各个角的子项名称
- `146kg` 的弱化展示

`146kg` 的展示原则：

- 必须保留
- 不能抢走雷达图的主视觉
- 可以作为低透明度大号背景数字，或作为不占空间的弱化装饰信息

设计原则：

- 雷达区成为右侧辅助分析区，但与左区共属一个整体盒子
- `146kg` 只能作为背景型或角标型信息存在，不再成为独立信息块
- 雷达区整体应比上一版更矮、更紧凑

## 4. 推荐布局方案

采用“同一外层盒子内，左区上下分层，右区紧凑雷达”的结构。

### 4.1 原因

相比上一版“卡片化摘要 + 卡片化雷达”，这次结构更贴近用户草图：

- 左区内部上下关系更清晰
- 雷达区不会再因为额外卡片边框和说明文案显得臃肿
- 整体更像监控大屏里的信息分栏，而不是后台卡片面板
- 与用户草图中的“左大区 + 右雷达区 + 中间分割线”更一致

### 4.2 视觉层级

从强到弱依次为：

1. 判级状态
2. 智能综合厚度
3. 智能级别分布
4. 右侧雷达区
5. 操作按钮

其中按钮重要但不应抢走摘要层的视觉焦点，因此被放在左区下层，以线性结构承接上层信息。

## 5. 实施约束

### 5.1 保持不变

- 综合结果区仍位于中间栏顶部
- 外层容器仍保持当前整体位置与主背景风格
- 按钮样式语义不变
- 颜色 token 不变
- 现有数据内容与交互回调不变

### 5.2 可调整内容

- 综合结果区内部 `grid` / `flex` 结构
- 信息块顺序和分组方式
- 雷达图容器尺寸
- 局部字体层级与间距
- 分割线布局
- `146kg` 的弱化呈现方式

## 6. 验收标准

完成后应满足：

- 左右两区位于同一个整体盒子中
- 左区内部明确分为“上方综合结果 / 下方按钮”
- `智能综合厚度` 显示为单一数字，例如 `6`
- `智能级别分布` 明确展示 `重1 20% / 重2 70%`
- 雷达图区域不再出现大面积说明文字和明细小块
- `146kg` 被保留，但不以独立抢眼卡片出现
- 三个按钮在左区下方单独成行
- 1920x1080 下综合结果区不溢出、不挤压、不换行异常

## 7. 验证方式

本次验证使用以下方式：

- `npm run lint`
- `npm run build`
- 使用 `playwright-cli` 在 `1920x1080` 视口下打开页面
- 通过截图检查综合结果区是否满足草图结构要求
- 检查浏览器控制台，避免新增布局相关错误

## 8. 结论

本次采用“同一外层盒子中，左区上方综合结果、左区下方按钮、右区紧凑雷达图”的重排方案。

这是当前最符合用户草图意图、同时对现有结构改动最小的方案，能够在不改变整体大屏框架的前提下，显著提升综合结果区的可读性，并去掉不符合大屏语境的卡片感。
