# 综合结果区排版调整设计说明

日期：2026-04-13  
项目：废钢判级监控大屏原型  
适用代码目录：`/Users/mac/Develop/work/yongyou/废钢判级监控大屏原型`

## 1. 背景

当前中间上方的综合结果区承载了以下信息：

- 判级状态
- 三个操作按钮
- 智能级别判断
- 智能扣杂雷达
- 扣杂总量与明细摘要

这些内容目前同时竞争同一行高，导致视觉重心分散，雷达图右侧说明信息过重，按钮与状态信息之间也缺少明确分层。

用户要求本次只重排该区域，不调整其他区域结构，并特别要求：

- 雷达图不要配太多文字
- 雷达图仅保留图本体与各角子项名称
- 判级状态、智能综合厚度、智能级别分布应放在上方一块
- 三个按钮单独放在下方
- 排版在当前容器内稳定，不出现溢出

## 2. 目标与非目标

### 2.1 目标

- 重新组织综合结果区的视觉层次
- 提高中上区域信息的扫读效率
- 让按钮区从信息摘要区中独立出来
- 降低雷达图的说明密度
- 保持整体风格、色彩语言、交互行为与周边区域一致

### 2.2 非目标

- 不改左侧栏和右侧分析栏
- 不改视频区和抓拍图区结构
- 不新增业务功能
- 不更改按钮行为
- 不修改整体三栏布局比例

## 3. 信息结构重组

综合结果区改为上下两层。

### 3.1 上层：状态摘要层

上层采用左右两列布局：

- 左列为主信息区
- 右列为扣杂雷达卡

#### 左列主信息区

左列承载 3 个核心摘要模块：

1. 判级状态
2. 智能综合厚度
3. 智能级别分布

其中：

- `判级状态` 继续作为最大字号主标题
- 车牌与当前监控点 badge 保留，但从属到状态信息下
- `智能综合厚度` 改为单独的大数字摘要，例如 `6`
- `智能综合厚度` 不显示单位，不展示厚度区间
- `智能级别分布` 仅负责展示候选级别和概率，例如：
  - `重1 20%`
  - `重2 70%`
- 级别分布继续保留条形概率条，但不再和“综合厚度”混排在同一视觉单元中

#### 右列扣杂雷达卡

右列保留一张更轻量的雷达卡。

雷达卡仅保留：

- 雷达图本体
- 图上各个角的子项名称
- 右上角一个小型总量摘要，例如 `146kg`

移除以下内容：

- 右侧大体量的扣杂明细列表
- “总量与各子项扣杂”辅助文案
- 过大的总量数字块

设计原则：

- 雷达卡成为“辅助分析块”，不再和左侧状态摘要争夺主视觉
- `146kg` 仅作为角标式信息，提醒存在扣杂总量，不再承担大标题角色

### 3.2 下层：操作按钮层

三个按钮独立成一整行，位于上层摘要之下：

- `结束判级`
- `异常处理`
- `中途离开`

布局原则：

- 按钮层与摘要层之间有明确分割
- 三个按钮保持同一基线
- 优先使用稳定的三列或自适应等宽布局
- 在默认大屏宽度下不换行

## 4. 推荐布局方案

采用“左主信息 + 右雷达卡 + 下排按钮”的结构。

### 4.1 原因

相比把所有摘要横向平铺，这种结构更适合当前容器宽高比例：

- 左侧状态信息需要更宽空间承载大标题和级别分布
- 雷达图作为图形信息，更适合固定在右侧独立卡片
- 按钮下置后，能避免与上方摘要争夺垂直空间
- 上下层关系更接近用户草图描述

### 4.2 视觉层级

从强到弱依次为：

1. 判级状态
2. 智能综合厚度
3. 智能级别分布
4. 扣杂雷达卡
5. 操作按钮

其中按钮重要但不应抢走摘要层的视觉焦点，因此单独下置最合理。

## 5. 实施约束

### 5.1 保持不变

- 综合结果区仍位于中间栏顶部
- 外层容器的边界、背景、分隔线风格不变
- 按钮样式语义不变
- 颜色 token 不变
- 现有数据内容与交互回调不变

### 5.2 可调整内容

- 综合结果区内部 `grid` / `flex` 结构
- 信息块顺序和分组方式
- 雷达图容器尺寸
- 局部字体层级与间距
- 说明文案精简

## 6. 验收标准

完成后应满足：

- 上方信息明确分为“左摘要 + 右雷达”
- `智能综合厚度` 显示为单一数字，例如 `6`
- `智能级别分布` 明确展示 `重1 20% / 重2 70%`
- 雷达图区域不再出现大面积说明文字和明细小块
- `146kg` 仅作为雷达区小摘要存在
- 三个按钮在下方单独成行
- 1920x1080 下综合结果区不溢出、不挤压、不换行异常

## 7. 验证方式

本次验证使用以下方式：

- `npm run lint`
- `npm run build`
- 使用 `playwright-cli` 在 `1920x1080` 视口下打开页面
- 通过截图检查综合结果区是否满足排版要求
- 检查浏览器控制台，避免新增布局相关错误

## 8. 结论

本次采用“上层状态摘要 + 右侧轻量雷达卡 + 下层按钮行”的重排方案。

这是当前最符合用户草图意图、同时对现有结构改动最小的方案，能够在不改变整体大屏框架的前提下，显著提升综合结果区的可读性与稳定性。
