# 卸料点总览大屏设计说明

项目：废钢判级监控大屏原型  
日期：2026-04-20  
适用目录：`/Users/mac/Develop/work/yongyou/废钢判级监控大屏原型`

## 目标

在现有项目的三屏切换体系中新增“卸料点总览大屏”，复刻 `卸料点总览原型.png` 的主内容区表达方式，同时明确不复刻原图顶部统计条，继续沿用项目现有顶栏与整体页面骨架。

## 设计约束

- 顶部导航继续使用现有项目的统一 `TopBarRegion`
- 新页面通过顶部下拉切换进入，作为第 3 个独立屏幕
- 主内容区必须遵循 `DESIGN.md`：
  - 以 near-black / white 为主
  - 语义色只做状态强调
  - 使用 Aeonik Pro/Inter 体系
  - 所有按钮/状态胶囊采用 pill 语言
  - 保持零阴影，仅靠对比和边界建立层次
- 原图中的“智能级别”字段统一改为“智能综合厚度”
- “智能综合厚度”取值范围为 1-10，使用假数据

## 结构

页面主体为深色监控场景，内部为四列卡片墙：

1. 卸料点卡片
   - 顶部监控画面区域
   - 右上角实时状态标签
   - 车牌与卸料点编号
   - 判级状态胶囊：判级中 / 已判级 / 无判级
   - 开始判级时间或“卸料点空闲中”提示
   - 底部两个指标块：
     - 智能综合厚度
     - 智能扣杂

2. 空闲卡片
   - 视频区改为占位监控空态
   - 元信息弱化
   - 继续保留指标块，但通过低对比呈现

## 数据映射

- 总览页使用独立假数据源，不复用单监控点数据
- 每张卡片字段：
  - `id`
  - `pointName`
  - `plate`
  - `status`
  - `startedAt`
  - `thicknessScore`（1-10）
  - `deductionWeight`
  - `isLive`
  - `imageSrc`
- 数据数量为 8 个卸料点，匹配原图的 4x2 浏览密度

## 组件边界

- `src/components/unloading-overview/constants.ts`
  - 存放卸料点总览假数据和状态映射
- `src/components/unloading-overview/UnloadingOverviewScreen.tsx`
  - 总览大屏主组件
  - 内含卡片墙与卡片子组件
- `src/components/dashboard/types.ts`
  - 扩展屏幕类型
- `src/components/dashboard/constants.ts`
  - 扩展顶部下拉屏幕列表

## 视觉策略

- 复刻原图的“工业监控卡片墙”感觉，但不照搬发光描边
- 卡片边框采用细线 + 局部语义色描边
- 车牌字号保持远距可扫读
- 指标块采用浅描边容器，保持分区感但避免后台表单味道
- “智能综合厚度”使用更直接的数值表达，避免旧版圆环等级语义干扰

- 保持构建可通过，不改动已有图像监控与排队监控逻辑
